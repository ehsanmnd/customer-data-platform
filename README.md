# پلتفرم داده مشتریان گروه صنعتی هوایار (CDP)

این پروژه یک **پلتفرم داده مشتریان (Customer Data Platform - CDP)** است که برای گروه صنعتی هوایار طراحی و توسعه داده شده است. هدف از این پلتفرم، ارائه یک داشبورد تحلیلی جامع برای بررسی رفتار مشتریان، سگمنتیشن (بخش‌بندی)، پیش‌بینی ریزش (Churn) و ارزش طول عمر مشتری (CLV) در قالب محیطی جذاب و کاربردی است.

## ویژگی‌های کلیدی

- **داشبورد تحلیلی جامع**: نمایش خلاصه‌ای از مهم‌ترین شاخص‌های کلیدی عملکرد (KPI) مانند کل مشتریان B2B، متوسط ارزش حیات (CLV)، نرخ ریزش و رشد ماهانه.
- **بخش‌بندی مشتریان (Segmentation)**: استفاده از مدل LRFM برای دسته‌بندی مشتریان به سگمنت‌های مختلف (مانند مشتریان وفادار، رویگردان، جدید و ...).
- **تجزیه و تحلیل ارزش طول عمر (CLV)**: بررسی مشتریان با بالاترین ارزش طول عمر و دسته‌بندی آنها بر اساس وضعیت ریسک.
- **پیش‌بینی و تحلیل ریزش (Churn Analysis)**: شناسایی مشتریان در معرض خطر ریزش برای انجام اقدامات پیشگیرانه.
- **رابط کاربری مدرن و واکنش‌گرا**: طراحی شده با استفاده از Tailwind CSS با پشتیبانی کامل از حالت تاریک (Dark Theme) و نمایش صحیح در دستگاه‌های مختلف.
- **نمودارهای تعاملی**: استفاده از کتابخانه Recharts برای نمایش بصری داده‌ها به صورت نمودارهای دایره‌ای، میله‌ای و ...

## تکنولوژی‌های استفاده شده

- **کتابخانه اصلی**: React 18
- **بیلد تول**: Vite
- **استایل‌دهی**: Tailwind CSS
- **آیکون‌ها**: Lucide React
- **نمودارها و بصری‌سازی داده‌ها**: Recharts
- **زبان برنامه‌نویسی**: TypeScript

## نحوه نصب و اجرای پروژه

برای اجرای این پروژه به صورت محلی در سیستم خود، مراحل زیر را دنبال کنید:

1. **کلون کردن مخزن**:
   ```bash
   git clone https://github.com/ehsanmnd/customer-data-platform.git
   cd customer-data-platform
   ```

2. **نصب وابستگی‌ها**:
   ```bash
   npm install
   ```

3. **اجرای سرور توسعه (Development Server)**:
   ```bash
   npm run dev
   ```
   پس از اجرای این دستور، برنامه در مرورگر شما در دسترس خواهد بود.

4. **بیلد پروژه برای محیط پروداکشن**:
   ```bash
   npm run build
   ```

## توسعه و استقرار (Deployment)

این پروژه به گونه‌ای پیکربندی شده است که با استفاده از **GitHub Actions** به صورت خودکار روی **GitHub Pages** مستقر (Deploy) می‌شود (فایل تنظیمات در `.github/workflows/deploy.yml` قرار دارد). با هر بار Push کردن تغییرات در شاخه `main`، نسخه جدید پروژه به صورت اتوماتیک پابلیش خواهد شد.
